Marketplace Agent Values Template for Solace Agent Mesh

Template for “Solace Agent Mesh and MCP Servers”

Use this template for the module “Solace Agent Mesh and MCP Servers”:

:::code{showCopyAction=true showLineNumbers=false language=shell} cat > ~/marketplace-agent-values.template « ‘EOF’

Standalone Agent Deployment Values for Marketplace Agent

This deploys a standalone agent to work with the existing agent-mesh deployment

deploymentMode: standalone agentId: marketplace-agent

Solace Broker Configuration

Using existing agent-mesh broker credentials

solaceBroker: url: “${SOLACE_BROKER_URL}” username: “solace-cloud-client” password: “${SOLACE_BROKER_PASSWORD}” vpn: “${SOLACE_BROKER_VPN}”

LLM Service Configuration

Using existing agent-mesh LLM service

llmService: generalModelName: “openai/${MODEL_NAME}” planningModelName: “openai/${MODEL_NAME}” endpoint: “http://${DATA_PLANE_LB}/bedrock-route” apiKey: “nokey”

Persistence Configuration

Use the existing PostgreSQL and SeaweedFS from agent-mesh deployment

persistence:

database: # Create a new database for marketplace agent in the existing PostgreSQL url: “postgresql+psycopg2://solace-agent-mesh-no-auth_orchestrator:solace-agent-mesh-no-auth_orchestrator@agent-mesh-postgresql.default.svc.cluster.local:5432/marketplace_agent”

s3: # Use the existing SeaweedFS deployment (S3-compatible) endpointUrl: “http://agent-mesh-seaweedfs.default.svc.cluster.local:8333” bucketName: “solace-agent-mesh-no-auth” # SeaweedFS credentials from agent-mesh-persistence secret accessKey: “solace-agent-mesh-no-auth” secretKey: “solace-agent-mesh-no-auth” region: “us-west-2”

Additional environment variables for the marketplace agent

env: NAMESPACE: “solace-agent-mesh-no-auth” MCP_MARKETPLACE_URL: “http://${DATA_PLANE_LB}/mcp-listener” MARKETPLACE_AGENT_DATABASE_URL: “postgresql+psycopg2://solace-agent-mesh-no-auth_orchestrator:solace-agent-mesh-no-auth_orchestrator@agent-mesh-postgresql.default.svc.cluster.local:5432/marketplace_agent”

Service Account (optional, can use default)

serviceAccount: create: false name: “default”

Resource limits

resources: requests: cpu: 100m memory: 256Mi limits: EOF :::
